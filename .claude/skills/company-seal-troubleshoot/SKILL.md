# 盖公司章异常排查

## 使用场景

当用户询问以下问题时，使用此技能：
- 合同盖公司章失败/异常
- 盖公司章出错
- 查询盖章相关的错误信息
- 排查盖章问题

**触发关键词**：盖公司章、盖章失败、盖章异常、公司章、PDF盖章

## 排查流程

### 第一步：提取合同号

从用户问题中提取**合同编号**（contract_code）。

**合同编号格式**：以字母 "C" 开头 + 数字，如 `C1776759658764987`

**处理规则**：
- ✅ 如果用户提供了合同编号 → 继续排查
- ❌ 如果用户未提供合同编号 → **必须反问用户**，要求提供合同编号

**反问示例**：
```
请提供需要排查的合同编号（格式如：C1776759658764987）
```

### 第二步：查询合同的 platform_instance_id

使用 `sre_query` 工具查询合同表，获取 `platform_instance_id` 字段：

```
sre_query(action="contract", contract_code="C1776759658764987")
```

**关注点**：
- 从返回结果中提取 `platform_instance_id` 字段
- 如果查询不到合同，提示用户检查合同编号是否正确

### 第三步：查询 FAST 日志

使用 `fast_log_query` 工具查询盖章日志，**queryString 使用 Lucene 语法**：

```
fast_log_query(
    queryString='("盖公司章PDF入参" or "盖公司章PDF出参") && "{platform_instance_id}"',
    size=20
)
```

**说明**：
- 将 `{platform_instance_id}` 替换为第二步获取的实际值
- 查询过去 15 小时的日志（默认）
- 如果需要特定时间段，可传入 `start_time` 和 `end_time`（格式：`YYYY-MM-DD HH:MM:SS`）

### 第四步：分析日志结果

分析 FAST 日志返回的结果：

**1. 去重日志**
- 合并相同时间段、相同内容的日志
- 保留入参和出参的配对关系

**2. 识别错误**
- 查找 `盖公司章PDF出参` 中的错误信息
- 提取错误码（如 `code=3001`）
- 提取错误描述（如 `未获取到有效的自动签授权信息`）

**3. 常见错误类型**

| 错误码 | 错误信息 | 可能原因 |
|--------|----------|----------|
| 3001 | 未获取到有效的自动签授权信息 | 装饰公司未配置自动签授权 |
| 3002 | 盖章失败 | PDF 文件异常或签章配置错误 |
| 其他 | 具体错误信息 | 根据错误信息分析 |

## 注意事项

1. **必须提供合同号**：如果用户未提供，必须反问，不能跳过
2. **日志去重**：展示结果时需要对重复日志进行去重
3. **入参出参配对**：展示时将入参和出参配对展示，便于理解
4. **时间范围**：默认查询过去 15 小时，如用户指定时间段则使用用户指定的
5. **Lucene 语法**：queryString 必须使用 Lucene 查询语法
